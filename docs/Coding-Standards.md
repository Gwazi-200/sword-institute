# Coding Standards

## Purpose
These standards help maintain consistency, readability, and long-term maintainability across the Sword Institute codebase.

## General Principles
- Write clear, readable, and intentional code
- Keep functions small and focused on a single responsibility
- Favor simple solutions over unnecessary complexity
- Document important logic where needed
- Maintain consistency across HTML, CSS, and JavaScript files

## HTML and Structure
- Use semantic HTML elements where appropriate
- Keep markup organized and easy to navigate
- Use meaningful class and id names
- Ensure forms and interactive elements are accessible

## CSS
- Use clear, descriptive class names
- Keep styles modular and grouped by purpose
- Favor readable formatting and consistent spacing
- Ensure responsive behavior for different screen sizes

## JavaScript
- Use clear variable and function names
- Prefer modern JavaScript syntax when appropriate
- Avoid duplication by reusing shared logic
- Handle errors gracefully and provide useful feedback to users

## File Organization
- Keep related logic together in dedicated files
- Separate authentication logic, UI behavior, and data access where practical
- Maintain a clean and predictable structure for future contributors

## Documentation
- Update documentation whenever significant features or workflows change
- Use clear headings and concise explanations in project docs
- Keep implementation notes aligned with the actual product behavior

## Review Expectations
- Pull requests should be easy to review and understand
- Changes should be tested in the browser where possible
- New features should preserve accessibility and usability standards
